# Ollama 集群扫描台

把原本的命令行扫描脚本改造成了一个可视化网页应用，主机地址可以在网页里动态添加/删除，
支持随时启动/停止扫描，扫描日志和结果实时显示。

## 功能

- **主机管理**：网页表单添加任意数量的 Ollama 地址（`http://ip:11434`），可随时删除，保存在持久化卷中，重启容器不丢失。
- **三阶段扫描**（逻辑与原脚本一致）：
  1. 发现各主机上的模型
  2. 快速可用性测试
  3. 高级代码生成测试（路径穿越防护 / 多方法类测试）
- **开始 / 停止按钮**：扫描可随时中断，不需要等它跑完。
- **实时日志**：网页轮询后端，滚动展示扫描过程。
- **结果面板**：按主机+模型分组展示每项测试的通过/失败情况。

## 在 Coolify 中部署

### 方式一：Docker Compose（推荐）

1. 在 Coolify 新建一个 **Resource → Docker Compose** 项目。
2. 把这个文件夹上传到你的 Git 仓库（或者用 Coolify 的「上传代码」功能），确保 `docker-compose.yml` 在根目录。
3. Coolify 检测到 `docker-compose.yml` 后会自动构建镜像。
4. 部署完成后，通过 Coolify 分配的域名/端口访问，默认容器内端口是 `8000`。
5. 数据卷 `ollama-scanner-data` 会持久化保存你添加的主机列表和最近一次扫描结果，重新部署不会丢失。

### 方式二：Dockerfile 直接构建

1. 在 Coolify 新建 **Resource → Application**，选择这个仓库，构建方式选 **Dockerfile**。
2. 设置端口映射：容器内 `8000` → 你想要的外部端口。
3. 挂载一个持久化卷到容器内的 `/data` 目录（Coolify 的 Storage / Volumes 设置里添加），否则主机列表在容器重建后会丢失。

### 环境变量

| 变量 | 说明 | 默认值 |
|---|---|---|
| `DATA_DIR` | 主机列表和扫描结果的存储目录 | `/data` |

## 本地测试（不用 Docker）

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

然后打开 `http://localhost:8000`。

## 使用说明

1. 打开网页后，在左侧「主机地址」面板输入 Ollama 地址（例如 `http://192.168.1.10:11434`），点击「+ 添加」。
2. 可以添加任意多个地址，每个都会显示在列表里，点击 × 可以移除。
3. 点击「▶ 开始扫描」启动三阶段扫描，日志会实时滚动显示。
4. 扫描过程中可以随时点击「■ 停止」中断。
5. 扫描全部完成（或被停止）后，下方「测试结果」面板会显示每个模型的通过情况。

## 关于安全

请只添加你自己拥有或已获得明确授权测试的 Ollama 服务器地址。这个工具会向目标主机发送真实的
推理请求（包括让模型生成代码并在扫描服务器本地执行测试），未经授权对他人服务器发起持续请求可能
违反相关服务条款或法律法规。
